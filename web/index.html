To view similarity results here, initiate the following steps:

  1. Begin by creating a folder on your PC. This will serve as the central location to manage all the files and results.
  2. For the assignment you're analyzing, download the section submissions archive. Ensure that you save this archive into the folder you've created in the previous step. It's vital to ensure your folder contains only the section submissions of the assignment you wish to check for plagiarism.
  3. Start the Plag Detector application.
  4. Enter the directory containing section submissions archives. This path serves as the location for storing the analysis results too.
  5. Specify the programming language used in the student assignments.
  6. Choose the desired operation:
    • Extract submissions and run MOSS for a thorough similarity check.
    • Extract submissions without executing MOSS for manual inspection.

Upon successful execution, a 'results' folder will be created in your specified directory. This folder will contain:

  • extracted_code_folders: Individual student folders with relevant programming files based on your language selection.
  • other_files: A compartmentalized directory with miscellaneous files (misc_files) and unzipped code files (unzipped_code_files).
  • archives: A directory preserving individual student submission archives.

About MOSS:

MOSS (Measure Of Software Similarity) is an automatic system designed to determine program similarity, primarily used in academia to detect plagiarism since 1994. While effective, it requires human intervention to conclude plagiarism cases.

Non-commercial Statement:

This application was developed for educational and academic purposes and is not intended for commercial use. It leverages the open-source capabilities of MOSS, honoring its non-commercial nature.
